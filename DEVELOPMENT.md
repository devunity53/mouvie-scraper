# 👨‍💻 Guide de Développement\n\n## Architecture\n\n```\nmouvie-scraper/\n├── launcher.py          # Point d'entrée avec auto-update\n├── mouvie_gui.py        # Orchestrateur principal\n├── config_manager.py    # Configuration (JSON)\n├── scraper.py           # Web scraping (BeautifulSoup)\n├── image_handler.py     # Gestion des images\n├── ui_builder.py        # Construction UI\n└── ui_components.py     # Composants réutilisables\n```\n\n## Pattern de conception\n\n### MVC-like Architecture\n- **Model**: `config_manager.py`, `scraper.py`\n- **View**: `ui_builder.py`, `ui_components.py`, `image_handler.py`\n- **Controller**: `mouvie_gui.py`\n\n### Separation of Concerns\nChaque module a une responsabilité unique:\n\n| Module | Responsabilité |\n|--------|----------------|\n| `launcher.py` | Lancement + Auto-update |\n| `config_manager.py` | Configuration JSON |\n| `scraper.py` | HTTP + parsing HTML |\n| `image_handler.py` | Téléchargement + affichage images |\n| `ui_builder.py` | Construction UI stateless |\n| `ui_components.py` | Composants UI réutilisables |\n| `mouvie_gui.py` | Orchestration |\n\n## Ajouter une nouvelle fonctionnalité\n\n### Exemple: Ajouter un filtre de résultats\n\n1. **Ajouter la classe** dans `ui_components.py`:\n```python\nclass FilterPanel:\n    def __init__(self, parent):\n        self.frame = ctk.CTkFrame(parent)\n        self.quality_var = ctk.StringVar()\n        # ...\n```\n\n2. **Ajouter le builder** dans `ui_builder.py`:\n```python\n@staticmethod\ndef build_filter_panel(parent):\n    return FilterPanel(parent)\n```\n\n3. **Intégrer dans** `mouvie_gui.py`:\n```python\nself.filter_panel = UIBuilder.build_filter_panel(torrent_tab)\nself.results = self.scraper.fetch_search_results(query)\nself.results = self.filter_panel.apply_filter(self.results)\n```\n\n## Cycle de mise à jour\n\n### 1. Développement local\n```bash\npython mouvie_gui.py  # Teste localement\n```\n\n### 2. Commit et push\n```bash\ngit add .\ngit commit -m \"feat: new feature\"\ngit push origin main\n```\n\n### 3. Update VERSION.txt\n```\n1.0.0 → 1.0.1 (patch)\n1.0.0 → 1.1.0 (minor)\n1.0.0 → 2.0.0 (major)\n```\n\n### 4. Les utilisateurs reçoivent l'update\nLe `launcher.py` détecte automatiquement la nouvelle version.\n\n## Code Style\n\n### Naming\n```python\n# Classes\nclass MouvieApp(ctk.CTk):\n    pass\n\n# Méthodes publiques\ndef perform_search(self):\n    pass\n\n# Méthodes privées\ndef _preload_detail_worker(self):\n    pass\n\n# Constantes\nDEFAULT_SITE = \"https://...\"\n```\n\n### Docstrings\n```python\nclass DetailPanel:\n    \"\"\"Manages the detail panel display\"\"\"\n    \n    def update(self, detail, item_detail_url):\n        \"\"\"Update the detail panel with new data.\"\"\"\n        pass\n```\n\n## Thread Safety\n\n```python\n# Utiliser des locks pour les opérations partagées\nwith self.cache_lock:\n    self.detail_cache[idx] = detail\n```\n\n## Gestion des erreurs\n\n```python\ntry:\n    response = requests.get(url, timeout=10)\n    response.raise_for_status()\nexcept requests.RequestException as e:\n    self.status_label.configure(text=f\"Erreur : {e}\")\n```\n\n## Testing\n\nPour tester localement sans GitHub:\n\n```python\n# Dans launcher.py, commenter:\n# updater.check_and_update()\n\n# Puis tester:\npython launcher.py\n```\n\n## Performance\n\n### Optimisations implémentées\n- ✅ Preloading des 10 premiers résultats\n- ✅ Cache des détails (évite re-scraping)\n- ✅ Chargement asynchrone des images\n- ✅ Thread-safe pour opérations longues\n\n### À optimiser\n- [ ] Cache persistant (SQLite)\n- [ ] Compression d'images\n- [ ] Pagination des résultats\n\n## Dépendances\n\n```\ncustomtkinter==5.2.0   # GUI moderne\nrequests==2.31.0       # HTTP\nbeautifulsoup4==4.12.2 # HTML parsing\nPillow==10.0.0         # Images\n```\n\n## Debugging\n\n### Logs détaillés\n```python\nimport logging\nlogging.basicConfig(level=logging.DEBUG)\nlogger = logging.getLogger(__name__)\nlogger.debug(\"Message de debug\")\n```\n\n### Inspection des données\n```python\nprint(f\"Results: {self.results}\")\nprint(f\"Cache: {self.detail_cache}\")\n```\n\n## Checklist avant release\n\n- [ ] Tests locaux réussis\n- [ ] Pas de print() de debug\n- [ ] Commentaires pertinents ajoutés\n- [ ] VERSION.txt mis à jour\n- [ ] CHANGELOG.md mis à jour\n- [ ] Commit + Push\n\n## Ressources\n\n- [CustomTkinter Docs](https://github.com/TomSchimansky/CustomTkinter)\n- [BeautifulSoup Docs](https://www.crummy.com/software/BeautifulSoup/bs4/doc/)\n- [Requests Docs](https://requests.readthedocs.io/)\n- [GitHub API](https://docs.github.com/en/rest)\n\n## Support\n\nPour les questions:\n1.
